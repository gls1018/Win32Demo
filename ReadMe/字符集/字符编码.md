

#### 分类

- 单字节字符集 SBCS
  - ASCII
  - 扩展ASCII
- 多字节字符集 MBCS
- 宽字符集 Unicode



#### 单字节字符集 SBCS

使用定长一个字节表示字符

- ASCII
  - 一个字节只使用7bit, 可以表示 128 种字符
- 扩展ASCII
  - 一个字节8bit全部使用，可以表示 256 种字符



#### 多字节字符集

多结字字符集中，英文字符用一个字符表示，利用连续多个扩展ASCII码的扩展区域来表示一个汉字。MBCS字符串由一个0字节结尾，这个0字节的数据类型是char

- GB2312
  - 每个汉字以两个字节表示
  - 简体中文编码范围是 0xA1A1 - 0xFEFE
  - 一共收录了6763个简体汉字， 628个字符， 包括拉丁字母，希腊字母、日本平假名和片假名等等
- GBK
  - 兼容GB2312标准，每个汉字以两个字节表示,加入了对繁体字的支持
- GB18030
  - 兼容GBK和GB2312标准。GB18030字符集采用单字节、双字节和四字节三种方式对字符编码。



缺陷: 多字节字符集中，不同国家对同一个字符的编码的值可能不一样，所以使用GBK编码的软件，到了日本，可能无法正常显示

#### unicode编码

- UTF-8
  - 可变长度编码方式，使用 1 ~ 4 个字节编码Unicode字符
  - 英文字符，UTF8和ASCII是相同的，只用1个字节
  - 大部分汉字采用3个字节编码，少量汉字采用4个字节
  - UTF8不存在字节序大小端问题
- UTF-16
  - 大部分汉字以2个字节存储，少部分汉字用4个字节存储
  - UTF16不兼容ASCII, 即使是英文字符，都是采用2个字节存储
  - Windows默认采用的是UTF-16
  - UTF-16编码有大端和小端之分，编码前会放置一个U+FEFF表示大端,或者U+FFFE来表示小端存储
- UTF-32
  - UTF-32对每个字符都用4字节
  - 有大端和小端之分，编码前用0xFEFF和0xFFFE区分
